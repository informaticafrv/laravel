Descripción del Proyecto

VideoTracker es una aplicación web desarrollada con Laravel 12 que permite a los usuarios gestionar su biblioteca personal de videojuegos. Cada usuario puede llevar un registro de los juegos que ha jugado, está jugando o tiene pendientes, puntuarlos, comentarlos y desbloquear logros asociados a cada título.

La aplicación combina un catálogo global compartido entre todos los usuarios con una biblioteca personal para cada uno. Los juegos del catálogo se pueden importar directamente desde la API oficial de IGDB (Internet Games Database), la base de datos de videojuegos de Twitch.

El proyecto implementa todos los requisitos mínimos del CRUD en Laravel y además incorpora características avanzadas como autenticación con Breeze, middleware propio, relaciones many-to-many con tabla pivot, integración con API externa mediante OAuth2 y reactividad en frontend con Alpine.js.

Funcionalidades principales

📚
Biblioteca Personal
Cada usuario gestiona su propia colección: añade juegos, les asigna nota, plataforma y estado (Pendiente, Jugando, Completado, Abandonado).
🌐
Catálogo Global
Base de datos compartida de juegos con búsqueda por título, filtrado por género, paginación y nota media calculada entre todos los usuarios.
🔍
Búsqueda IGDB en Tiempo Real
Al crear un juego, el formulario busca en la API de IGDB según el usuario escribe, mostrando resultados con portada y género sin recargar la página.
🏆
Sistema de Logros
Cada juego genera automáticamente 20 logros (Bronce, Plata, Oro, Platino) al crearse. Los usuarios pueden marcarlos como desbloqueados.
💬
Comentarios
Sistema de comentarios por juego en la vista de detalle, con usuario y fecha de publicación.
📊
Dashboard de Estadísticas
Página de inicio con resumen de la colección del usuario: total de juegos, jugando actualmente y completados.

Estructura de la Base de Datos

El proyecto cuenta con 6 tablas propias más las de Laravel (cache, jobs, password_resets). Las relaciones entre ellas modelan que un usuario puede tener muchos juegos en su biblioteca y que un mismo juego puede pertenecer a muchos usuarios con datos distintos para cada uno.

users id, name, email, password, remember_token, timestamps games id, titulo (unique), genero, portada, portada_url, igdb_id, timestamps videogames id, user_id → users, game_id → games, plataforma, puntuacion_personal, estado, timestamps comments id, user_id → users, game_id → games, contenido, timestamps achievements id, game_id → games, nombre, descripcion, imagen_url, timestamps achievement_user id, user_id → users, achievement_id → achievements, timestamps Relaciones: User ──< videogames >── Game (many-to-many con datos en pivot) User ──< achievement_user >── Achievement (many-to-many) Game ──< comments (one-to-many) Game ──< achievements (one-to-many)

Tabla pivot videogames

Es el núcleo del proyecto. No es solo una tabla de enlace: almacena los datos personales de cada usuario para cada juego (nota, plataforma y estado). Esto permite que dos usuarios tengan el mismo juego con notas y estados distintos.

Rutas y Middleware

Todas las rutas de la aplicación están protegidas con el middleware auth de Laravel. Las rutas de edición y eliminación añaden además un middleware propio que verifica la propiedad del recurso.

// Grupo principal: requiere autenticación
Route::middleware(['auth'])->group(function () {

    Route::get('/dashboard', ...)->name('dashboard');
    Route::get('/biblioteca',  [VideogameController::class, 'index'])->name('videogames.index');
    Route::get('/catalogo',    [VideogameController::class, 'catalogo'])->name('videogames.catalogo');
    Route::post('/votar/{game_id}', [VideogameController::class, 'votar'])->name('videogames.votar');
    Route::get('/juegos/{id}', [VideogameController::class, 'show'])->name('games.show');

    // Subgrupo: requiere además ser propietario del juego
    Route::middleware(['game.owner'])->group(function () {
        Route::get('/videojuegos/{id}/editar', ...)->name('videogames.edit');
        Route::put('/videojuegos/{id}',        ...)->name('videogames.update');
        Route::delete('/videojuegos/{id}',     ...)->name('videogames.destroy');
    });
});

Middleware CheckGameOwner

Middleware propio registrado como game.owner. Comprueba que el user_id del registro coincida con el usuario autenticado antes de permitir editar o eliminar:

public function handle(Request $request, Closure $next): Response
{
    $videojuego = Videogame::find($request->route('id'));

    if ($videojuego && $videojuego->user_id !== Auth::id()) {
        abort(403, 'Acceso denegado: Este videojuego no te pertenece.');
    }

    return $next($request);
}

Gracias a este middleware es imposible que un usuario edite o borre el juego de otro aunque conozca la URL directa, sin necesidad de comprobar esto en cada controlador individualmente.

Características Destacadas de Eloquent ORM

firstOrCreate — evitar duplicados en el catálogo

Cuando un usuario añade un juego, se busca en la tabla global por título. Si no existe se crea; si ya existe se reutiliza. Así el catálogo no tiene duplicados aunque dos usuarios añadan el mismo título.

$game = Game::firstOrCreate(
    ['titulo' => $validated['titulo']],
    ['genero' => $validated['genero'], 'portada_url' => $request->portada_url]
);

updateOrCreate — upsert en la biblioteca personal

Si el usuario ya tiene ese juego en su biblioteca, actualiza los datos. Si no, lo crea. Evita la lógica manual de buscar primero y decidir después.

Videogame::updateOrCreate(
    ['user_id' => Auth::id(), 'game_id' => $game->id],
    ['plataforma' => $validated['plataforma'], 'estado' => $validated['estado']]
);

->when() — filtros condicionales encadenados

Aplica un where solo si la condición es verdadera. Si el usuario no ha escrito en el buscador, ese filtro no se añade a la consulta.

Game::query()
    ->when($buscar, fn($q) => $q->where('titulo', 'LIKE', "%{$buscar}%"))
    ->when($genero, fn($q) => $q->where('genero', $genero))
    ->paginate(6)
    ->withQueryString();

Eager loading ->with() — evitar el problema N+1

Carga todas las relaciones necesarias en una sola consulta, evitando que Laravel haga una query por cada comentario para buscar su autor.

$juego = Game::with(['comments.user', 'achievements'])->findOrFail($id);

toggle() en relación many-to-many

Marca o desmarca un logro para el usuario en la tabla pivot achievement_user de forma automática, sin comprobar si existe previamente.

$user->achievements()->toggle($id);

Integración con la API de IGDB

IGDB (Internet Games Database) es la base de datos oficial de videojuegos de Twitch. La aplicación la integra para permitir buscar y autocompletar juegos reales con su portada y género.

Autenticación OAuth2

Antes de cada búsqueda se solicita un token de acceso temporal a los servidores de Twitch usando las credenciales de la aplicación:

$response = Http::post('https://id.twitch.tv/oauth2/token', [
    'client_id'     => config('services.igdb.client_id'),
    'client_secret' => config('services.igdb.client_secret'),
    'grant_type'    => 'client_credentials',
]);
$token = $response->json()['access_token'];

Consulta a la API

Con el token obtenido se realiza la búsqueda usando el cliente HTTP de Laravel:

Http::withHeaders([
    'Client-ID'     => config('services.igdb.client_id'),
    'Authorization' => 'Bearer ' . $token,
])->withBody("search \"{$search}\"; fields name, cover.url, genres.name, id; limit 5;", 'text/plain')
  ->post('https://api.igdb.com/v4/games');

Búsqueda en tiempo real con Alpine.js

El formulario de creación de juegos muestra resultados mientras el usuario escribe, sin recargar la página, usando Alpine.js para hacer la petición al endpoint interno:

// En la vista create.blade.php
<div x-data="{ query: '', results: [] }">
    <input @input.debounce.500ms="fetchGames()" x-model="query" />

    <template x-for="game in results">
        <div @click="selectGame(game)" x-text="game.name"></div>
    </template>
</div>

El debounce.500ms espera a que el usuario deje de escribir durante medio segundo antes de lanzar la petición, evitando llamadas innecesarias a la API por cada letra.

Factories y Seeders

Para facilitar el desarrollo y las pruebas, el proyecto incluye factories y un seeder que generan datos realistas con un solo comando (php artisan db:seed).

DatabaseSeeder

Crea un usuario de prueba, 10 juegos con 20 logros cada uno, y los relaciona todos al usuario:

// Crea usuario de prueba
$user = User::factory()->create([
    'email' => 'test@example.com',
    'password' => bcrypt('password'),
]);

// Crea 10 juegos, cada uno con 20 logros asociados
$juegos = Game::factory(10)
    ->has(Achievement::factory()->count(20), 'achievements')
    ->create();

// Relaciona cada juego al usuario con datos aleatorios
foreach ($juegos as $juego) {
    Videogame::create([
        'user_id'            => $user->id,
        'game_id'            => $juego->id,
        'plataforma'         => collect(['PC', 'PS5', 'Xbox', 'Switch'])->random(),
        'puntuacion_personal'=> rand(5, 10),
        'estado'             => collect(['Jugando', 'Completado', 'Pendiente'])->random(),
    ]);
}

GameFactory

Genera juegos con títulos únicos, géneros aleatorios de una lista predefinida e IDs de IGDB únicos:

return [
    'titulo'    => $this->faker->unique()->words(3, true),
    'genero'    => $this->faker->randomElement(['RPG', 'Acción', 'Aventura', 'Shooter', 'Terror', 'Indie']),
    'portada_url' => 'https://placehold.co/600x800?text=Game+Cover',
    'igdb_id'   => $this->faker->unique()->numberBetween(100, 999999),
];
